# Improve Drag-and-Drop Visual Feedback

## Problem
`DragOverlay` renders `{null}` — no visual ghost when dragging. Original card just fades to 0.4 opacity, which is too subtle.

## Fix

### File: `frontend/src/features/board/components/KanbanBoard.tsx`

1. Add `activeId` state to track the currently dragged item
2. Set `activeId` on `onDragStart`, clear on `onDragEnd`/`onDragCancel`
3. Render a styled ghost card inside `<DragOverlay>` — a copy of the task card with elevated shadow and slight rotation for depth
4. The original card in the list stays visible but dimmed (keep existing opacity 0.4)

### File: `frontend/src/features/board/components/TaskColumn.tsx`

5. Add a drop indicator line when `isOver` is true — a colored bar at the top of the column or between cards to show where the task will land

## Verification
- `npm run lint`
- Visual: dragging a task shows a floating ghost card with shadow
- Visual: dropping zone highlights with a colored indicator
