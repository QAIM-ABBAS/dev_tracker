# Plan: Add GitHub CI/CD to Linux Server

## Goal
Add a GitHub Actions workflow that automatically deploys to a Linux server on push to `main` via SSH + Docker Compose.

## Approach
- **Trigger**: Push to `main` branch (manual dispatch also supported)
- **Deploy method**: SSH into server, pull latest code, `docker compose up --build -d`
- **Auth**: SSH private key stored as GitHub secret
- **Server prereqs**: Docker + Docker Compose already installed

## Files to Create

### 1. `.github/workflows/deploy.yml`
GitHub Actions workflow that:
- Triggers on `push to main` and `workflow_dispatch` (manual)
- Checks out the repo on the runner
- Uses `appleboy/ssh-action` to SSH into the server
- On the server: `cd /opt/projectflow && git pull origin main && docker compose up --build -d`
- Cleans up old images after deploy

### 2. `deploy/.env.production` (template, not committed with secrets)
A production `.env` template showing what needs to exist on the server at `/opt/projectflow/.env`. Includes:
- Strong PostgreSQL password
- Production CORS origins (server IP/domain)
- Real webhook secret
- Real JWT secret key

## GitHub Secrets Required (user must set these)
| Secret Name | Description |
|---|---|
| `DEPLOY_HOST` | Server IP or hostname (e.g. `123.45.67.89`) |
| `DEPLOY_USER` | SSH username (e.g. `root` or `deploy`) |
| `DEPLOY_KEY` | SSH private key (full PEM content) |
| `DEPLOY_PATH` | Remote path to the repo (e.g. `/opt/projectflow`) |

## Workflow Behavior
1. On push to `main` → SSH into server → git pull → docker compose up --build -d
2. On manual trigger → same flow
3. Optional: skip deploy if commit message contains `[skip deploy]`

## Key Details
- The server's `.env` file must be pre-configured (the workflow does NOT manage secrets)
- `docker compose up --build -d` handles rebuilds and restarts automatically
- Old dangling images are pruned after each deploy
- The workflow uses `appleboy/ssh-action@v1.2.0` (widely used, well-maintained)

## Verification
1. Push the workflow file to `main`
2. Set the 4 GitHub secrets in repo Settings → Secrets → Actions
3. Ensure server has the repo cloned at the configured `DEPLOY_PATH` with a valid `.env`
4. Trigger the workflow manually from Actions tab or push a commit to `main`
5. Check the Actions log for successful SSH + deploy output
6. Verify the app is running: `curl http://<server-ip>:3000` (frontend) and `http://<server-ip>:8000/docs` (backend)
